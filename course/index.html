The Dopamine Machine

Every tap of "Open Pack" sets off a chain reaction across three computers — and it all finishes before you finish blinking. Here is how.

What is Dopamaxxing?

Dopamaxxing is a Pokémon card gacha game: spend virtual coins to open packs of cards, collect rare Pokémon, and show off your collection. It is the excitement of pulling a holographic Charizard, recreated entirely in software.

The whole app is built on three pillars — opening, selling, and battling. Everything else supports these three loops.

🎴
Open Packs

Spend coins, get random Pokémon cards. Pull odds vary by rarity — common cards fall out often, legendary ones almost never.

💰
Sell & Trade

Every card has a buyback value based on condition. Sell duplicates to earn more coins and keep the loop going.

⚔️
Battle Trainers

Use your card collection to fight AI trainers in turn-based combat. Your card stats determine the outcome.

💡
Key insight

The whole app is one big request-response loop: the browser asks, the server decides, the database remembers.

The Moment of the Click

Think of the app like a vending machine. When you tap "Open Pack," you trigger a chain reaction across multiple computers. Here is every step, in order.

1
You click "Open Pack" in your browser

Your finger taps a button rendered by React. Nothing has happened on the server yet.

2
Your browser sends a request to the server

The browser fires a POST request to /api/open-pack over HTTPS.

3
Server checks: Are you logged in? Got enough coins? Bag not full?

The authentication check happens first. If any condition fails, the server sends back an error and stops — no cards are given.

4
Server fetches cards from the database + stats from PokeAPI simultaneously

Using Promise.all(), three database queries and an external API call fire at the exact same time.

5
Server rolls the random card selection, calculates stats and worth

The RNG runs on the server — never the browser — so players cannot cheat by manipulating the odds.

6
Server deducts coins, saves cards to database

Both writes happen inside a single database transaction — ensuring you can never lose coins without getting cards.

7
Server sends cards back — your browser plays the reveal animation

The server returns a JSON response with the card data; the browser then runs the animation to reveal your new cards.

The director rule

Code ↔ English: What Auth Actually Means

Every single API route starts the same way — it checks who you are before doing anything. Here is what that looks like in real code from app/api/open-pack/route.ts.

CODE 
export async function POST(request: NextRequest) {
    try {
        const supabase = await createClient()
        const {
            data: { user },
        } = await supabase.auth.getUser()

        if (!user)
            return NextResponse.json(
                { error: 'Unauthorized' },
                { status: 401 }
            )
PLAIN ENGLISH
This function wakes up every time someone sends a POST request to /api/open-pack.
Connect to Supabase using the login cookies from the request.
Ask Supabase: who sent this? Are they logged in?
Get back the user object — their ID, email, and auth status.
If nobody is logged in (no valid cookie) — stop right here.
Send back an error: "Unauthorized." Code 401 is the universal internet code for this — every browser in the world understands it.
🚪
Authentication middleware

This pattern — checking identity before doing anything — is called authentication middleware. Every serious app does this on every protected endpoint. It is the server equivalent of a bouncer checking IDs — no valid ID, no entry.

💡
Why this matters for directing AI

When you ask an AI to add a new feature to an API route, you need to tell it: "put the new logic after the auth check." If you forget, the AI might generate code that runs before the user is verified — a serious security hole that is hard to spot.

export function pickRarityFromWeights(
    weights: Record<string, number>
): string {

    const total = Object.values(weights)
        .reduce((a, b) => a + b, 0)

    let roll = Math.random() * total

    for (const [rarity, weight] of
         Object.entries(weights)) {
        roll -= weight
        if (roll <= 0) return rarity
    }

    return 'Common'
}
PLAIN ENGLISH
This function picks a rarity using a weighted lottery. Give it a table of weights, get a rarity back.
Add up all the weights to find the total size of the lottery space. (Example: 78 + 16 + 4.5 + … = ~100)
Roll a random number anywhere from 0 to that total — this is the dart thrown at the lottery board.
Walk through each rarity one by one, from top to bottom of the list.
Subtract that rarity's weight from the roll — like crossing off lottery numbers.
The moment the roll hits zero or below — that rarity wins the lottery and gets returned.
If nothing hit (floating point rounding edge case) — fall back to Common as a safe default.

Why every gacha game has this

This pattern — guaranteed reward after N attempts — is used by nearly every gacha game ever made. Game designers call it a soft pity or mercy system. It exists because humans have a powerful emotional reaction to feeling cheated by pure randomness — the math makes the game feel fair even when you are unlucky.

Rolling Real Stats

Once PokeAPI sends back base stats, this function in lib/pokemon-stats.ts turns them into battle-ready numbers:

CODE 
export function rollStats(
    rarity: string,
    baseStats?: BaseStats
): CardStats {
    const mult = RARITY_STAT_MULTIPLIER[rarity] ?? 1.0
    const variance = () => 0.85 + Math.random() * 0.3

    return {
        stat_atk:   Math.round((baseStats?.attack    ?? randBase()) * mult * variance()),
        stat_def:   Math.round((baseStats?.defense   ?? randBase()) * mult * variance()),
        stat_spatk: Math.round((baseStats?.specialAtk ?? randBase()) * mult * variance()),
        stat_spdef: Math.round((baseStats?.specialDef ?? randBase()) * mult * variance()),
        stat_spd:   Math.round((baseStats?.speed     ?? randBase()) * mult * variance()),
        stat_accuracy: Math.round(75 * mult * variance()),
        stat_evasion:  Math.round(75 * mult * variance()),
    }
}
PLAIN ENGLISH

This function calculates a card's 7 battle stats from its Pokémon's real base stats.

Get the rarity multiplier — Epic = 2.0×, Celestial = 2.8×, Common = 0.8×.

Create a random variance: anywhere from 0.85 to 1.15 (±15%) — so no two cards are identical.

Attack = real Charizard base attack ?? random fallback if PokeAPI was down...

...multiplied by rarity boost, then by random variance. Round to a whole number.

Accuracy and evasion have no real base stat, so they start from 75 and scale with rarity.

💡
The ?? Fallback Pattern

baseStats?.attack ?? randBase() — the ?? makes the whole system fault-tolerant. If PokeAPI goes down, pack opening keeps working with random stats. This pattern — always having a fallback — is one of the most important habits in production software.

The Damage Formula

Every attack runs through this function in lib/n-battle.ts. It takes the attacker, the move, and the defender — and returns one number: how much HP to remove.

Example: Flamethrower (90) × Fire→Grass (2.0×) × SPATK bonus (1.63×) × DEF reduction (0.74×) × no crit (1.0×) = ~217 damage
CODE 
export function calculateDamage(
    attacker: BattleCard,
    move: Attack,
    defender: BattleCard,
): number {
    const base = move.damage ?? 0
    const typeBonus = getTypeEffectiveness(
        move.type, defender.pokemon_type ?? 'Normal'
    )
    const statBonus = move.damage_class === 'special'
        ? attacker.stat_spatk / 100
        : attacker.stat_atk / 100
    const defMult = move.damage_class === 'special'
        ? 1 - (defender.stat_spdef / 300)
        : 1 - (defender.stat_def / 300)
    const crit = Math.random() < 0.0625 ? 1.5 : 1
    return Math.max(1, Math.round(
        base * typeBonus * statBonus * defMult * crit
    ))
}
PLAIN ENGLISH

How much damage does this move deal?

Base power of the move — Flamethrower = 90, Scratch = 40.

Look up the type matchup — Fire vs Grass = 2.0×, Fire vs Water = 0.5×.

Special moves use Special Attack; physical moves use regular Attack.

The defender's Defense or Special Defense reduces incoming damage.

6.25% chance of a critical hit — 1.5× damage, no warning.

Final damage is always at least 1. You can never deal zero damage.

Type Effectiveness & Status Conditions

Type matchups are the rock-paper-scissors of the battle system. Green = 2× damage (super effective). Pink = 0.5× (not very effective).
