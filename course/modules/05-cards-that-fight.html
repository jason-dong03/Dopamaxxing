The Damage Formula

Every attack runs through this function in lib/n-battle.ts. It takes the attacker, the move, and the defender — and returns one number: how much HP to remove.

Example: Flamethrower (90) × Fire→Grass (2.0×) × SPATK bonus (1.63×) × DEF reduction (0.74×) × no crit (1.0×) = ~217 damage
CODE 
export function calculateDamage(
    attacker: BattleCard,
    move: Attack,
    defender: BattleCard,
): number {
    const base = move.damage ?? 0
    const typeBonus = getTypeEffectiveness(
        move.type, defender.pokemon_type ?? 'Normal'
    )
    const statBonus = move.damage_class === 'special'
        ? attacker.stat_spatk / 100
        : attacker.stat_atk / 100
    const defMult = move.damage_class === 'special'
        ? 1 - (defender.stat_spdef / 300)
        : 1 - (defender.stat_def / 300)
    const crit = Math.random() < 0.0625 ? 1.5 : 1
    return Math.max(1, Math.round(
        base * typeBonus * statBonus * defMult * crit
    ))
}
PLAIN ENGLISH

How much damage does this move deal?

Base power of the move — Flamethrower = 90, Scratch = 40.

Look up the type matchup — Fire vs Grass = 2.0×, Fire vs Water = 0.5×.

Special moves use Special Attack; physical moves use regular Attack.

The defender's Defense or Special Defense reduces incoming damage.

6.25% chance of a critical hit — 1.5× damage, no warning.

Final damage is always at least 1. You can never deal zero damage.

Type Effectiveness & Status Conditions

Type matchups are the rock-paper-scissors of the battle system. Green = 2× damage (super effective). Pink = 0.5× (not very effective).

💧 Water 2× 0.5× 0.5× 1× 1×

The useBattle Hook: One File to Run It All

All battle state — which phase, whose turn, HP values, animations — lives in a single custom hook called useBattle. Here's what it manages:

CODE 
export function useBattle(onClose: () => void) {
    const [phase, setPhase] = useState<BattlePhase>('pre-dialogue')
    const [battle, setBattle] = useState<BattleState | null>(null)
    const [acting, setActing] = useState(false)

    // ... all battle logic here ...

    return {
        phase, battle, acting,
        battleMenu, cards, selected,
        doAttack, doSwitch, retry,
        getBattleText,
    }
}
PLAIN ENGLISH

useBattle is a custom hook — a reusable bundle of all battle state and logic.

phase tracks which screen is active: pre-dialogue, team-reveal, battle, won, etc.

battle holds the full game state — synced from the database after every turn.

acting prevents double-clicks: set to true while the server is calculating, false when done.

Return everything the battle screen needs: data to display + functions to call on input.

The screen itself just calls doAttack() — the hook handles all the API calls and updates.
