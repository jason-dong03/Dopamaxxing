Talking to the Outside World

When your card shows ATK: 84 / DEF: 62, those numbers didn't come from your database. They came from a free Pokémon data service, got classified by an AI, and landed in your card in under 2 seconds.

Most real apps are just clever orchestrations of a dozen outside services — each a specialist you hire through a formal contract called an API.

The Four Outside Services

Dopamaxxing doesn't do everything itself. It hires four specialists — each one a world-class tool you'd spend months building from scratch.

🌍

PokeAPI

Free. No API key needed. Provides real base stats and learnable moves for all 1,000+ Pokémon. Responses are cached for 24 hours.

🤖

Groq AI

Runs Llama 3.1. Given a move's name and effect text, it returns structured JSON: damage class, heal percentage, status effects. Used once per move when a card is created.

🗄️

Supabase

PostgreSQL in the cloud. Stores everything: user profiles, owned cards, battle states, quests, achievements. Also handles login via Google/Discord OAuth.

💳

Stripe

Real money transactions. Users buy coin packages. Stripe handles the checkout page, payment processing, and notifies the app when payment succeeds via a webhook.

Rolling Real Stats

Once PokeAPI sends back base stats, this function in lib/pokemon-stats.ts turns them into battle-ready numbers:

CODE 
export function rollStats(
    rarity: string,
    baseStats?: BaseStats
): CardStats {
    const mult = RARITY_STAT_MULTIPLIER[rarity] ?? 1.0
    const variance = () => 0.85 + Math.random() * 0.3

    return {
        stat_atk:   Math.round((baseStats?.attack    ?? randBase()) * mult * variance()),
        stat_def:   Math.round((baseStats?.defense   ?? randBase()) * mult * variance()),
        stat_spatk: Math.round((baseStats?.specialAtk ?? randBase()) * mult * variance()),
        stat_spdef: Math.round((baseStats?.specialDef ?? randBase()) * mult * variance()),
        stat_spd:   Math.round((baseStats?.speed     ?? randBase()) * mult * variance()),
        stat_accuracy: Math.round(75 * mult * variance()),
        stat_evasion:  Math.round(75 * mult * variance()),
    }
}
PLAIN ENGLISH

This function calculates a card's 7 battle stats from its Pokémon's real base stats.

Get the rarity multiplier — Epic = 2.0×, Celestial = 2.8×, Common = 0.8×.

Create a random variance: anywhere from 0.85 to 1.15 (±15%) — so no two cards are identical.

Attack = real Charizard base attack ?? random fallback if PokeAPI was down...

...multiplied by rarity boost, then by random variance. Round to a whole number.

Accuracy and evasion have no real base stat, so they start from 75 and scale with rarity.

💡
The ?? Fallback Pattern

baseStats?.attack ?? randBase() — the ?? makes the whole system fault-tolerant. If PokeAPI goes down, pack opening keeps working with random stats. This pattern — always having a fallback — is one of the most important habits in production software.

Groq: AI as a Data Transformer

When a card learns a move, the app needs structured data — but PokeAPI returns plain English text. Groq bridges that gap.

INPUT — PokeAPI text "Flamethrower: The target is hit with a fiery blast.
It may also leave the target with a burn."
↓ Groq AI (Llama 3.1)
OUTPUT — Structured JSON {
  "power": 90,
  "type": "fire",
  "damage_class": "special",
  "burn_chance": 10,
  "priority": 0
}
🧠
AI as a Quiet Data Transformer

This is one of the most practical uses of LLMs in real products: not a chatbot, but a silent converter that turns messy text into clean data that code can use. No human had to manually classify 900+ Pokémon moves.
